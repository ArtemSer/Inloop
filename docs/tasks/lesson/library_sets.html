Inloop / Tasks / Library with Sets

Collections II

This task and Collections I (Library with Lists) may have much in common, but in this task, you will solve certain aspects in another way. As a result of that, you will get to know other parts of the Java API.

Look at the given design model for a library!

You can open image (click on it) to copy text from the image.

Task

Implement the design model with the following constraints! You can reuse parts of the code from the other task if you have already solved it.

  1. Use the Set interface for the aggregation between Library and Book.
  2. As concrete Set type, use either HashSet or TreeSet. Write the code in such a way that you have to make as few changes as possible when it comes to switching the concreteSet type! Think of polymorphism to reach this goal!
  3. Use iterators to access the elements of the sets used.
  4. Add the method listStockByAuthor() to the class Library. This method returns a Map containing every author associated with a Set of books by the respective author. Use the generic interface java.util.Map<K,V> to implement this behavior.
  5. Use the Comparable interface to make a Book comparable by its isbn.
  6. Think about advantages and disadvantages of the data structures used in this task in comparison to lists.
  7. Think about the advantages of an implementation with sets over an implementation with arrays!
Hints

The interface Comparable is a member of the Java API. For more information about it, consult the Java API Documentation.
